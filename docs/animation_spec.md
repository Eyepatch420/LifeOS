# Animation Spec

Every animation in the app, its trigger, and the duration/curve from `core/animations/` it uses. Updated as animations are implemented.

**Philosophy:** animation must communicate state, not decorate. If removing an animation wouldn't reduce usability or clarity, it shouldn't exist. All new/changed animations respect Reduce Motion (`MediaQuery.of(context).disableAnimations`) — see the Reduce Motion column.

| Animation | Trigger | Widgets | Duration | Curve | Reduce Motion fallback | Notes |
|---|---|---|---|---|---|---|
| Splash logo reveal | App cold start | `SplashLogoAnimation` | `AppDurations.slow` (~450ms) + hold | `AppCurves.easeOutCubic` | — | Fade + scale + gradient reveal, then tagline fade |
| Onboarding illustration hand-off | Page index change | `OnboardingIllustrationMorph` | `AppMotionPresets.onboardingIllustration` (500ms) | `AppCurves.easeOutCubic` | Plain crossfade, no translate/scale | Outgoing shrinks to 92%, **fully exits** to the left (pixel-based `Transform.translate` across the full available width via `LayoutBuilder`, not a fractional `SlideTransition`), fades to 0.4. Incoming starts fully off-screen right at 85% scale, slides to center, grows to 100%. Both animate simultaneously — reads as "passing one object to another," not a hard cut. |
| Onboarding title/subtitle transition | Page index change | `_OnboardingTextSwitcher` (private, in `onboarding_screen.dart`) | `AppMotionPresets.section` (300ms) | `AppCurves.easeOutCubic` | Plain crossfade, no slide | Fade + slight upward slide (`Offset(0, 0.15) → Offset.zero`), independent of the illustration's timing/keying |
| Onboarding per-page accent | Page index change | `OnboardingScreen` (root `TweenAnimationBuilder<OnboardingAccent>`) | `AppMotionPresets.section` (300ms) | `AppCurves.easeOutCubic` | — (color-only lerp, no motion) | Drives the button color, active progress dot color, and a subtle background gradient tint. Uses `OnboardingAccentTween` (mirrors `AppColorsExtension.lerp()`'s style). Scoped entirely to `OnboardingScreen`'s subtree — never touches the app-wide `ThemeData`/`AnimatedTheme` pipeline, which stays reserved for workspace/dark-mode theming (see `theme.md`). |
| Onboarding progress dots | Page index change | `ProgressDots` | `AppDurations.fast` | `AppCurves.easeOutCubic` | — | Active dot width/opacity animates independently; accepts an optional `color` override (used by onboarding's per-page accent) |
| Workspace theme switch | Bottom nav tap, or setup-screen accent live preview | Root `AnimatedTheme` | `AppDurations.medium` | `AppCurves.easeOutCubic` | — | Backed by `AppColorsExtension.lerp()` — gradients/FAB/chips interpolate, not snap. Never rebuilds the whole app: only widgets that read `Theme.of(context)`/`context.appColors` repaint. |
| Bottom nav pill slide | Tab tap | `FloatingBottomNav` | `AppDurations.medium` | spring-ish curve | — | `AnimatedAlign`/`AnimatedPositioned` |
| Bottom nav icon morph | Tab tap | `FloatingBottomNav` icons | `AppDurations.fast` | `AppCurves.easeOutCubic` | — | Scale + outline/filled swap |
| Home section entrance | Home screen first build | `StaggeredEntrance` wrapping `FadeSlideIn` | `AppDurations.medium` per item, staggered | `AppCurves.easeOutCubic` | — | Applies to each of the 7 registered Home sections as `HomeScreen` renders them in `homeSectionsProvider` order |
| Home hero entrance | Home screen first build | `HomeHeroSection` (`FadeSlideIn` around the content column) | `AppMotionPresets.hero` (300ms) | `AppCurves.easeOutCubic` | — (no scale/slide beyond the base fade+slide-up) | Only the greeting/date/actions/banner column animates in — the SVG/gradient backdrop stays static/instant since it's a backdrop, not "content" |
| Home hero time-of-day tint cross-fade | Time-of-day bucket changes during a session | `GradientHeroBackground` (`MorphSwitcher`) | `AppDurations.medium` (via `MorphSwitcher`'s default) | default `FadeThroughTransition` | — (color-only cross-fade) | Keyed on `TimeOfDayTint.bucket` (a stable enum), not the `greeting` string, so a future locale/copy change can't silently break the animation key |
| Timeline per-step entrance | Section first renders | `FadeSlideIn` per step, `Stagger.delayForIndex` (80ms/step) | `AppMotionPresets.timeline` | `AppCurves.easeOutCubic` | — | Fixed a real bug: previously `StaggeredEntrance` wrapped the whole step `Row` as ONE child, so the stagger never actually staggered per-step; now each `_TimelineStepWidget` fades in individually |
| Habit streak ring | Value change | `AnimatedStreakRing` | `AppMotionPresets.section` (300ms) | `AppCurves.easeOutCubic` | — | Tweens progress value; routed through the named preset (was inline `AppDurations.medium`/`AppCurves.easeOutCubic`) |
| Empty state entrance | A section's data list is empty | `EmptyState` (`FadeSlideIn`) | `AppDurations.medium` (default) | `AppCurves.easeOutCubic` | — | Same entrance treatment as populated content, so switching between empty/populated never feels like a different kind of screen |
| Page transitions | Route change | `page_transitions.dart` (`buildFadeThroughPage`) | `AppDurations.page` (~500ms) | `AppCurves.easeOutCubic` | — | `FadeThroughTransition` via `CustomTransitionPage`. Applied to splash/onboarding/user-setup routes **and** the Home shell branch's `GoRoute` (via an explicit `pageBuilder`) — a bare `StatefulShellRoute` has no `pageBuilder` of its own and would otherwise fall back to the platform default transition for the `/user-setup` → `/home` hop. |
| Button press/release | Tap down/up | `PressableScale` | `AppDurations.fast` | `AppCurves.easeOutCubic` | Skips the scale, tap still registers | Used by `PrimaryButton`, `CircularActionButton`, nav icons |
| Button hover (desktop) | Mouse enter/exit | `PrimaryButton` (`MouseRegion` + `AnimatedScale`/shadow) | `AppDurations.fast` | `AppCurves.easeOutCubic` | Keeps the shadow/color signal, skips the scale lift | No-op on touch platforms (`MouseRegion` never fires there) |
| Button loading crossfade | `isLoading` toggles | `PrimaryButton` (`AnimatedSwitcher`) | `AppDurations.fast` | default | — | Label ↔ `CircularProgressIndicator` |
| Button disabled state | `onPressed`/`isLoading` state change | `PrimaryButton` (`AnimatedOpacity`) | `AppDurations.fast` | `AppCurves.easeOutCubic` | — | Distinct animated dim (0.45 opacity), not just Material's default disabled styling — reads clearly different from both loading and enabled |

Decorative-only additions (e.g. a hero particle/sparkle overlay) are out of scope unless proven to cost nothing — see the Performance section of `implemented_features.md`/`future_work.md`. Visual polish never takes priority over performance or responsiveness.
