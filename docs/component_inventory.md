# Component Inventory

Detailed per-widget reference. `widget_tree.md` stays a lighter-weight status index; this file is the durable home for widget documentation as the app grows past a couple dozen widgets. Seeded with the widgets touched or added during the onboarding/setup refinement pass — extend this table as new widgets land.

| Widget | Purpose | Dependencies | Animations | Consumers |
|---|---|---|---|---|
| `OnboardingAccent` / `OnboardingAccentTween` | Per-onboarding-page accent bundle (primary color + gradient), with a custom `Tween` so it can be driven by `TweenAnimationBuilder` | `flutter/material.dart` only | N/A (it's the animatable value, not a widget) — consumed by the accent `TweenAnimationBuilder` in `OnboardingScreen` | `OnboardingScreen`, `OnboardingPage` (as a required field) |
| `OnboardingIllustrationMorph` | Choreographs the outgoing/incoming onboarding illustration hand-off (full pixel-based exit/entry, not fractional) | `LottieIllustration`, `AppMotionPresets` | Scale (1→0.92 outgoing, 0.85→1 incoming), opacity (1→0.4 outgoing, 0→1 incoming), pixel-based `Transform.translate` slide across the full available width via `LayoutBuilder`; Reduce-Motion fallback: plain crossfade | `OnboardingScreen` |

## How to add an entry

When a widget is added or meaningfully changed:
1. Add/update its row here with Purpose, Dependencies, Animations, Consumers.
2. Add/update its row in `widget_tree.md` (status only: implemented/planned).
3. If it introduces a new animation category, add it to `animation_spec.md`'s table.
